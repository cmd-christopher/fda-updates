# Phase 5 Plan 02: CSS Refinements for Readability

**Phase:** 05-detail-readability
**Plan:** 02
**Subsystem:** ui
**Depends on:** Plan 05-01 (format_pi_text rewrite producing new HTML structure)

## Goal

Make detail page text visually scannable by tuning typography, spacing, and visual hierarchy so that sub-section headings, indication lists, and body paragraphs have clear differentiation and adequate breathing room.

## Context

The UI review (03-UI-REVIEW.md) scored:
- Typography: 2/4 (26 distinct font sizes, subsection headings too small at 0.95rem, warning content smaller than body)
- Spacing: 3/4 (section-content horizontal padding too tight at 0.25rem)

Plan 05-01 fixes the *structure* (headings, lists, paragraph breaks). This plan fixes the *visual presentation*.

## Tasks

### Task 1: Sub-section heading prominence

**Files:** site/css/custom.css

**What:**
1. Increase `.pi-subsection` font-size from `0.95rem` → `1.05rem` (must be visibly larger than body text)
2. Add a left accent border to `.pi-subsection`: `border-left: 3px solid var(--orchid)` + `padding-left: 0.75rem`
3. Increase top margin from `1.25rem` → `1.5rem` for more visual separation
4. In dark mode, keep the border visible with the orchid-dark variant

**Why:** Sub-section headings (e.g., "1.1 Plaque Psoriasis") are the primary navigation aid inside long PI sections. At 0.95rem they're barely distinguishable from body text, especially when Plan 01 starts producing many more of them.

### Task 2: Section content padding and line length

**Files:** site/css/custom.css

**What:**
1. Increase `.section-content` horizontal padding from `0.25rem` → `1rem` (left/right) so text isn't flush with section edges
2. Set explicit `font-size: 0.95rem` on `.section-content` for consistency
3. Increase `line-height` from `1.7` → `1.75` for medical text readability
4. Reduce `.section-content` max-width from `850px` → `800px` for better line length (80-90 chars ideal for reading)

### Task 3: Warning content sizing fix

**Files:** site/css/custom.css

**What:**
1. Increase `.warning-content` font-size from `0.9rem` → `0.95rem` — warning content should never be smaller than body text

### Task 4: Indication lead-in styling

**Files:** site/css/custom.css

**What:**
1. Style `.indication-lead` with `font-weight: 600` and `margin-bottom: 0.5rem` to set up the bullet list that follows
2. Ensure `.pi-list` items with long text wrap properly (no overflow issues)

## Execution

**Wave 1:** All 4 tasks are CSS-only changes, can execute sequentially in one commit.

**Duration estimate:** 10 min

## Success Criteria

1. Sub-section headings have a visible left accent border and are at least 1.05rem
2. Section content text has 1rem horizontal padding (breathing room)
3. Warning content is 0.95rem (not smaller than body)
4. Indication lead-in sentence is visually distinct from the bullet list that follows
5. All changes look correct in both light and dark mode
