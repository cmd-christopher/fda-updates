  <action>
1. **Read the existing `build.py` fully** to understand its current structure before modifying it.

2. **Add a `sanitize_html(text)` function** near the top of `build.py` (after imports, before `DATA_PATH`):
   - Use Python's `re` module (no external dependencies)
   - Strip dangerous content: `<script>...</script>`, `<style>...</style>`, `<iframe>...</iframe>` — remove these tags AND their content entirely
   - Strip event attributes: `onclick`, `onerror`, `onload`, `onmouseover`, `onfocus`, `onblur`, `onsubmit`, and any other `on*` attributes — remove the attribute only, keep the tag
   - Strip `style` attributes — remove the attribute only, keep the tag
   - Preserve structural/safe tags: `<table>`, `<tr>`, `<td>`, `<th>`, `<thead>`, `<tbody>`, `<p>`, `<br>`, `<br/>`, `<ul>`, `<ol>`, `<li>`, `<b>`, `<i>`, `<strong>`, `<em>`, `<h1>`-`<h6>`, `<sub>`, `<sup>`, `<div>`, `<span>`, `<a>` (without event/style attributes)
   - The function accepts a string, strips dangerous content, returns a `Markup` object from Jinja2's `markupsafe.Markup` class so it's marked safe for Jinja2 rendering
   - Import `markupsafe.Markup` at the top
   
3. **Add slug collision detection** after reading drugs from JSON but before rendering:
   - Track seen slugs in a dict: `slug_counts = {}`
   - For each drug, if the slug already exists in `slug_counts`, append `-{digits_from_application_number}` (extract just the numeric digits from `application_number`, e.g., "NDA123456" → "123456")
   - Update `drug["slug"]` with the unique slug and increment the counter
   - If no collision, leave the slug as-is (already unique from `fda_approvals.py`)

4. **Add detail page rendering** in the `main()` function after the index page rendering:
   - Register `sanitize_html` as a Jinja2 filter: `env.filters["sanitize_html"] = sanitize_html`
   - Get the `drug_detail.html` template (will be created in Plan 02; for now, create a minimal placeholder template at `templates/drug_detail.html` that extends `base.html` and renders `{{ drug.brand_name }}` as the title so `build.py` doesn't error)
   - For each drug, create the output directory `site/drugs/` with `os.makedirs`, then render the template with `drug=drug` and write to `site/drugs/{drug['slug']}.html`
   - The index page rendering must remain unchanged — do not modify the existing `index.html` rendering code

5. **Create a minimal placeholder template** at `templates/drug_detail.html` so `build.py` can run:
