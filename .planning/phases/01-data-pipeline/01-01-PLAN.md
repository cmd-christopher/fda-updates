</objective>

    **1a. Add SUPPL fetching function:**
    Add a new function `fetch_suppl_approvals(date_from, date_to, limit=100)` that:
    - Queries `drugsfda.json` with search terms: `submissions.submission_type:SUPPL+AND+submissions.submission_status:AP`
    - Paginates through results (same pattern as existing `fetch_drugsfda_approvals`)
    - For each entry, finds the SUPPL submission(s) with efficacy-related `submission_class_code_description` values

    - Post-filters by the SUPPL submission's actual `submission_status_date` within the date range (NOT the drug's max submission date — this avoids the sort pitfall described in PITFALLS.md and AGENTS.md)

    **1b. Deduplication logic:**
    When combining ORIG and SUPPL results, each entry in the final list represents a distinct *approval event*. A drug like KEYTRUDA will have one ORIG entry (type_badge: "New Drug") and potentially multiple SUPPL entries (type_badge: "New Indication") at different dates — these are separate rows, not duplicates. However, two entries with the same application_number AND same approval_date AND same type are true duplicates and must be removed.

    **1c. Add type_badge field:**
    - For ORIG submissions with `submission_class_code_description` containing "New Molecular Entity" or "Type 1": `type_badge = "New Drug"`
    - For ORIG submissions with other classifications (e.g., "New Combo", "New Formulation"): `type_badge = "New Drug"` (they're still original approvals)
    - For SUPPL submissions with efficacy classification: `type_badge = "New Indication"`
    - Include `type_badge` in each drug record output

    **1d. Add indication_preview field:**
    Add a function `truncate_indication(text, max_length=100)` that:
    - Takes the `indications_and_usage` field from the label data
    - Strips HTML tags (simple regex strip, not full sanitization — that's Phase 3)
    - Truncates at the first sentence boundary (period followed by space or end) or the first clause boundary (comma or semicolon after 50+ chars), whichever comes first
    - Falls back to character truncation at max_length if no sentence boundary is found
    - Appends "..." if truncated
    - Returns the truncated string, or the drug's `submission_class_code_description` as fallback if label is None
    - Compute `indication_preview` for each drug and include it in the output

    **1e. Add slug field:**
    Add a function `slugify(name)` that:
    - Lowercases, strips whitespace, normalizes Unicode to ASCII, replaces non-word chars, collapses hyphens/spaces
    - Returns a URL-safe slug from `brand_name` (preferred) or `generic_name`
    - Include `slug` in each drug record output

    **1f. Update CLI and main():**

    - The `main()` function should call both `fetch_drugsfda_approvals()` (ORIG) and `fetch_suppl_approvals()` (SUPPL) when `--type all` (default), combine and deduplicate results, then enrich with labels
    - Preserve the `--skip-labels` flag behavior
    - Preserve the `--from`, `--to`, `--limit`, and `--output` flags unchanged
