    Create the build pipeline that reads the JSON data contract and produces static HTML.

    **1a. Create templates/base.html:**
    Minimal shared layout with:
    - `<!DOCTYPE html>` with `lang="en"` and `data-theme="light"` on `<html>`
    - `<head>` with viewport meta, title block `{% block title %}FDA Drug Approvals{% endblock %}`
    - `<body>` with `<main>{% block content %}{% endblock %}</main>`
    - A simple header: `<header><h1>FDA Drug Approvals</h1><p>Last updated: {{ last_updated }}</p></header>`
    - A simple footer: `<footer><p>Data from <a href="https://api.fda.gov/">openFDA</a></p></footer>`
    - NO CSS framework yet (that's Phase 2). Just basic semantic HTML.
    - Jinja2 autoescaping is handled in build.py, not in the template

    **1b. Create templates/index.html:**
    Extends base.html. Contains:
    - A `<table>` with columns: Approval Date, Type, Brand Name, Generic Name, Category, Indication
    - Rows populated from `drugs` variable via `{% for drug in drugs %}`
    - Each row links the brand name to `drugs/{{ drug.slug }}.html` (detail pages are Phase 3)
    - Type badges displayed as plain text: "New Drug" or "New Indication"
    - Empty-state handling: "No drug approvals found for this period." message if `drugs` list is empty
    - Data-sort attributes on date cells for future List.js integration (Phase 2): `<td data-sort="{{ drug.approval_date }}">{{ drug.approval_date_formatted }}</td>`
    - Display dates in human-readable format (e.g., "Apr 17, 2026") while keeping sortable ISO dates in data attributes

    **1c. Create build.py:**
    A Python script (~80-120 lines) that:
    ```python
    #!/usr/bin/env python3
    """Build the FDA drug approvals static site from JSON data."""
    import json
    import sys
    import os
    from datetime import datetime
    from jinja2 import Environment, FileSystemLoader, select_autoescape
    ```

    Key implementation:
    1. Read `data/approvals.json` — if missing or malformed, print a clear error and exit with non-zero status
    2. Validate required fields in each drug record (brand_name or generic_name, approval_date, application_number, type_badge, slug). If a required field is missing, print a specific error identifying the drug and field, then exit with non-zero status.
    3. Create Jinja2 environment with `FileSystemLoader('templates')` and `select_autoescape(['html', 'xml'])`
    4. Add a custom Jinja2 filter `format_date` that converts "YYYY-MM-DD" to "Mon DD, YYYY" (e.g., "2026-04-17" → "Apr 17, 2026")
    5. Render `index.html` template with `drugs=data['drugs'], last_updated=datetime.now().strftime('%B %d, %Y')`
    6. Write `site/index.html` (create `site/` directory if it doesn't exist, but do NOT overwrite `.git/` or other files in site/)
    7. Print a summary: "Built site/index.html with X drug approvals"

    **1d. Handle edge cases:**
    - If `data/approvals.json` has `count: 0` and `drugs: []`, render the "No drug approvals found" message
    - If a drug has `label: null`, the template still renders correctly (indication preview comes from `indication_preview` field which has a fallback)
    - If `pharm_class_epc` is an empty list, display "Classified" or the first MOA class, or "—" as a last resort. Never show a blank cell.

    **1e. Install Jinja2 if needed:**
    ```bash
    pip3 install --user jinja2
    ```

    **1f. Test the build:**
    ```bash
    python3 build.py
    ```
    Then verify `site/index.html` exists and contains drug data from the JSON.
