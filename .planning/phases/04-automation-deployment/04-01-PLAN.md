Purpose: Weekly automation requires fast, efficient runs. Per D-02 and D-03, incremental label fetching via set_id cache reduces API calls dramatically for routine weekly updates where most drugs are unchanged.

  **3. Add set_id to label output** — in fetch_label(), after extracting label fields, add:

  This extracts the set_id from the openFDA label API response (the `id` field in each label result). The openFDA label API returns each result with an `id` field that IS the set_id UUID (e.g., "f3e35b01-17a4-4ca7-8667-4447e80c29a5"). When fetch_label returns None (404 or error), set_id should be absent from the drug dict entirely.

  **4. Add load_previous_approvals() function:**
  ```python
  def load_previous_approvals(path="data/approvals.json"):
      """Load previous run's approvals.json to reuse label data for cached drugs."""
      if not os.path.exists(path):
          return {}
      try:
          with open(path) as f:
              data = json.load(f)
          # Build app_num → drug mapping for label reuse
          prev = {}
          for drug in data.get("drugs", []):
              app_num = drug.get("application_number", "")
              if app_num and drug.get("label"):
                  prev[app_num] = drug
          return prev
      except (json.JSONDecodeError, OSError):
          return {}
  ```

  **5. Add save_label_cache() function:**
  ```python
  def save_label_cache(drugs, cache_path):
      """Save app_num → set_id mapping for future incremental runs."""
      cache = {}
      for drug in drugs:
          app_num = drug.get("application_number", "")
          label = drug.get("label")
          if app_num and label and label.get("set_id"):
              cache[app_num] = label["set_id"]
      with open(cache_path, "w") as f:
          json.dump(cache, f, indent=2)
  ```

  **6. Modify the label fetching loop in main()** to support caching:

  When `args.cache` is True:
  - Before the label fetching loop, call `previous_data = load_previous_approvals()`
  - In the loop, before calling fetch_label(), check if the drug's `application_number` exists in `previous_data` AND has label data (not None). If so, reuse that label data directly, print a message to stderr like `"[{i}/{total}] {name} (cached)"`, and skip the API call. Set `drug["label"]` to the previous label dict.
  - For drugs NOT in the previous data, or with None labels, call fetch_label() normally and print `"[{i}/{total}] {name}..."` as before.
  - After all drugs are processed (at the end of main, before writing output), call `save_label_cache(drugs, LABEL_CACHE_PATH)` to write the cache file.

  When `args.cache` is False (default):
  - Behavior is identical to current script. Do NOT call load_previous_approvals or save_label_cache.
